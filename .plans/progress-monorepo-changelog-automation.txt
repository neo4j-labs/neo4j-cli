Migrated changie from a single-project config to a multi-project setup with

- .changes/v*.md (deleted — 15 versioned files removed from root)

Key Decisions:
- changie's `ProjectConfig` struct does NOT support per-project `changesDir` — it uses root `changesDir` + `key` as subdirectory for versioned files (confirmed from source at changie@v1.21.0/core/utils.go:GetAllVersions)
- The `ProjectConfig` field for changelog path is `changelog` (not `changelogPath`) — using the wrong field name silently fails
- All unreleased change files share `.changes/unreleased/` (root level); per-project unreleased subdirs are not used by changie — cascade in task-004 must duplicate files with different `project:` field values, not copy between directories
- `changie latest --project aura-cli` returns `aura-cliv1.7.0` (project key prepended) — downstream workflows need to strip the prefix or use `--remove-prefix`
- `ProjectsVersionSeparator` defaults to empty string; tasks 004/005 should set it to `-` if cleaner output is needed
