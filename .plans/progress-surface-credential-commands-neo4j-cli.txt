Created `neo4j-cli/aura/credential.go` in the `aura` package exporting `NewCredentialCmd`.
The command tree has `credential` as the root with four subcommands (`add`, `list`, `remove`,
`use`), each containing an `aura-client` leaf command that delegates directly to

Files Changed:
- neo4j-cli/aura/credential.go (created: NewCredentialCmd with full aura-client subcommand tree)

- Each action verb (`add`, `list`, `remove`, `use`) is an intermediate command with a single
  `aura-client` subcommand, establishing the pattern for future credential providers.
- Flag definitions for `add aura-client` mirror those in the internal credential/add.go exactly
  (`--name`, `--client-id`, `--client-secret`, all required).
- Internal helper functions are unexported (lowercase) since only `NewCredentialCmd` needs to
  be public — the internal builders are implementation details.
