TASK-002: Extract registerOutputFlag helper and update output.go
Date: 2026-05-01T01:00:00Z
================================================================================

Summary:
Created RegisterOutputFlag function in neo4j-cli/aura/flags.go (exported, in the
aura package which both binary entry points already import). The function registers
--output as a PersistentFlag on the given command and installs a PersistentPreRunE
hook that validates the value against ValidOutputValues and binds it to cfg.Global.
output.go was already updated to cfg.Global.Output() in task-001 so no change needed.

Files Changed:
- neo4j-cli/aura/flags.go (created: RegisterOutputFlag helper function)

Key Decisions:
- Placed RegisterOutputFlag in the aura package (not a new internal package) since
  both neo4j-cli/main.go and neo4j-cli/aura/cmd/main.go already import this package,
  avoiding a new dependency cycle.
- Function is exported (PascalCase) so entry-point main packages can call it.
- PersistentPreRunE uses cmd.Flags().Lookup("output") — cobra's Flags() includes
  inherited persistent flags, so leaf command context finds the root-registered flag.

TASK-003: Wire EnableTraverseRunHooks and registerOutputFlag into both binary entry points
Date: 2026-05-01T02:00:00Z
================================================================================

Summary:
Called aura.RegisterOutputFlag on the root command in neo4j-cli/main.go NewCmd(),
and called aura.RegisterOutputFlag on the standalone aura-cli root in
neo4j-cli/aura/cmd/main.go. Set cobra.EnableTraverseRunHooks = true before
cmd.Execute() in both main() functions so the root-level PersistentPreRunE
validation hook fires for every subcommand even when child PersistentPreRunE
hooks are present.

Files Changed:
- neo4j-cli/main.go (modified: added RegisterOutputFlag call in NewCmd, EnableTraverseRunHooks in main)
- neo4j-cli/aura/cmd/main.go (modified: added RegisterOutputFlag call and EnableTraverseRunHooks in main)

Key Decisions:
- cobra.EnableTraverseRunHooks is a package-level global, set before Execute() in main()
- RegisterOutputFlag is called in NewCmd() (not main()) for neo4j-cli so the flag is
  available to tests that call NewCmd() directly; EnableTraverseRunHooks stays in main()
  since it's a binary-level concern
- pflag's AddFlagSet silently ignores duplicate-named flags (child's flag shadows parent's),
  so the root's --output flag is temporarily shadowed by the resource-group subcommands'
  own --output registrations until task-004 removes them; the root PersistentPreRunE still
  correctly reads and binds the resolved flag value via cmd.Flags().Lookup("output")
